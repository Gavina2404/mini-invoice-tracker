## The task

Build a small app that lets a user manage invoices for one company: create, list, view, edit and delete invoices, and see the computed totals including VAT. Mauritius standard VAT is 15% — use that as the default. All amounts are in Mauritian Rupees (MUR).

## Tech stack — your choice (JavaScript preferred)

- Use whatever stack you're most comfortable and productive in — a frontend, a backend, and a database.
- JavaScript / TypeScript is preferred: we build with React/Next.js, Node.js/Express and MongoDB/Postgres, so a JS/TS stack helps us gauge fit — but shipping a clean, working app matters more than the exact tools.
- Whatever you choose, tell us what you picked and why in the README.
- Version control: Git, with an incremental commit history (not one big final commit).

## Core requirements (must-have)

1. Create an invoice via a form that supports at least one line item.
2. List all invoices in a table: invoice number, client, issue date, total (incl. VAT), status.
3. View a single invoice showing subtotal, VAT amount and grand total.
4. Edit and delete an invoice.
5. Compute totals (subtotal, VAT, total) on the BACKEND — never trust amounts sent by the client.
6. Basic validation: required fields, unique invoice number, positive quantities/prices.
7. Document how to run it (frontend / backend / DB), the required env vars, the choices you made, and what you'd do with more time.

## Suggested data model — `Invoice`

| Field | Type | Notes |
|---|---|---|
| `id` | string / number | Generated by your database (e.g. UUID, ObjectId, auto-increment). |
| `invoiceNumber` | string | Unique, e.g. INV-2026-001. |
| `clientName` | string | Required. |
| `issueDate` | date | Required. |
| `lineItems` | array | Each: { description, quantity, unitPrice }. At least one. |
| `vatRate` | number (%) | Default 15 (Mauritius). |
| `status` | enum | draft | sent | paid. |

## Sample data & a self-check

`sample-invoices.json` holds raw invoice inputs (no id, no totals — your backend computes those). As a quick sanity check, **INV-2026-001** should compute to:

- Subtotal: `10 × 2500 + 1 × 12000` = **37,000**
- VAT (15%): **5,550**
- Total: **42,550**
